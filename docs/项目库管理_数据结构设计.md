# 项目库管理系统 - 详细设计方案

## 一、系统概述

本方案用于设计项目库管理系统，支持：
- 项目创建、编辑、发布
- 设备管理与绑定
- 课程绑定（带版本）
- 项目导入/导出（加密解密）
- 项目版本管理
- 用户权限管理

---

## 二、实体数据结构

### 2.1 项目库基本信息

```cpp
struct Project {
    int id;                           // 项目ID
    QString name;                     // 项目名称
    QString description;              // 项目描述
    QString status;                   // 状态: draft/published
    
    // 版本管理
    QString current_version;          // 当前版本: "1.0.0"
    int library_project_id;          // 对应项目库ID
    QString source;                   // 来源: local/library
    
    // 权限
    int owner_id;                    // 创建人ID
    
    QDateTime created_at;
    QDateTime updated_at;
    
    QList<ProjectDevice> devices;    // 绑定的设备列表
    
    QVariantMap reserved;            // 保留字段
};
```

### 2.2 绑定设备的基本信息

```cpp
struct ProjectDevice {

```

## 三、用户权限管理

### 3.1 用户

```cpp
struct User {

    QString role;                      // admin/operator/viewer
```

### 3.2 权限定义

```cpp
enum class PermissionType {
    PROJECT_CREATE,                   // 创建项目
    PROJECT_EDIT,                     // 编辑项目
    PROJECT_DELETE,                   // 删除项目
    PROJECT_EXPORT,                   // 导出项目
    PROJECT_IMPORT,                   // 导入项目
    DEVICE_MANAGE,                    // 设备管理
    COURSE_MANAGE,                    // 课程管理
    VERSION_MANAGE                    // 版本管理
};
```

### 3.3 权限矩阵

| 角色 | 创建 | 编辑 | 删除 | 导出 | 导入 | 设备 | 课程 | 版本 |
|-----|-----|-----|-----|-----|-----|-----|-----|-----|
| 管理员 | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| 操作员 | ✓ | ✓ | ✗ | ✓ | ✓ | ✓ | ✓ | ✗ |
| 查看员 | ✗ | ✗ | ✗ | ✗ | ✗ | ✗ | ✗ | ✗ |

### 3.4 权限继承

```
权限检查流程:
用户登录
    ↓
获取用户角色
    ↓
检查角色权限
    ↓
项目权限 = 角色权限 (继承)
    ↓
允许操作?

特殊规则:
├── 项目的创建者始终拥有全部权限
├── 管理员可以分配项目访问权限给其他角色
└── 查看员只能查看，不能操作
```

## 四、功能设计

### 4.1 新建项目

```
项目创建流程:
1. 权限检查 (PROJECT_CREATE)
       ↓
2. 输入项目名称和描述
       ↓
3. 选择要绑定的设备
       ↓
4. 为每个设备分配课程(指定版本)
       ↓
5. 保存项目
       ↓
6. 自动生成版本 v1.0.0
```

### 4.2 绑定设备和课程

```
设备绑定关系:
项目
├── 设备A (CR400AF-Z)
│   ├── 课程1 (v1.0)
│   └── 课程2 (v1.1)
├── 设备B (CRH380A统)
│   └── 课程1 (v1.0)
└── 设备C (CR400BF)
    └── 课程3 (v2.0)
```

### 4.3 设备管理逻辑

```
设备查找逻辑:
├── 新建项目时
│   ├── 从设备资源库查找同名设备
│   ├── 找到 → 绑定
│   └── 未找到 → 提示注册或创建
│
└── 导入项目时
    ├── 必须连接设备资源库
    ├── 验证设备是否存在
    │   ├── 存在 → 绑定
    │   └── 不存在 → 自动注册设备 → 绑定
    └── 继续处理课程数据
```

## 五、导入/导出设计

### 5.1 加密解密设计

#### 5.1.1 加密流程

```
┌─────────────────────────────────────────────────────────────┐
│                    导出加密流程                              │
├─────────────────────────────────────────────────────────────┤
│  1. 准备项目数据                                            │
│       ↓                                                     │
│  2. 收集项目+设备+课程数据                                  │
│       ↓                                                     │
│  3. 打包每个课程的数据包                                      │
│       ↓                                                     │
│  4. 生成项目JSON                                           │
│       ↓                                                     │
│  5. 生成随机AES密钥 (256位)                                 │
│       ↓                                                     │
│  6. 使用AES密钥加密数据                                     │
│       ↓                                                     │
│  7. 使用RSA公钥加密AES密钥                                  │
│       ↓                                                     │
│  8. 打包加密数据到文件                                      │
│       ↓                                                     │
│  9. 输出加密文件                                            │
└─────────────────────────────────────────────────────────────┘
```

#### 5.1.2 解密流程

```
┌─────────────────────────────────────────────────────────────┐
│                    导入解密流程                              │
├─────────────────────────────────────────────────────────────┤
│  1. 登录验证 + 权限检查 (PROJECT_IMPORT)                    │
│       ↓                                                     │
│  2. 选择加密文件                                            │
│       ↓                                                     │
│  3. 使用RSA私钥解密AES密钥                                  │
│       ↓                                                     │
│  4. 使用AES密钥解密数据                                     │
│       ↓                                                     │
│  5. 验证解密后的数据完整性                                   │
│       ↓                                                     │
│  6. 解析项目数据                                            │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 导出包结构

#### 5.2.1 非加密导出

```
project_export_001.zip
├── project.json                      // 项目信息
├── devices.json                      // 设备信息
└── courses/                         // 课程数据包
    ├── course_001/
```

```
└── data.enc                         // AES加密的项目数据+课程数据包
```

## 六、版本管理

### 6.1 版本标识

```cpp
// 语义化版本: MAJOR.MINOR.PATCH
// - MAJOR: 重大变更(结构变化)
// - MINOR: 新增功能(兼容)
// - PATCH: Bug修复
```

### 6.2 版本管理数据表

```sql
-- 项目表
CREATE TABLE projects (
    id INTEGER PRIMARY KEY,
    name VARCHAR(255),
    description TEXT,
    status VARCHAR(20) DEFAULT 'draft',
    current_version VARCHAR(20) DEFAULT '1.0.0',
    library_project_id INTEGER,
    source VARCHAR(20) DEFAULT 'local',
    owner_id INTEGER,
    created_at DATETIME,
    updated_at DATETIME,
    reserved TEXT
);

-- 设备资源库
CREATE TABLE device_registry (
    id INTEGER PRIMARY KEY,
    name VARCHAR(255),
    type VARCHAR(50),
    code VARCHAR(100),
    ip_address VARCHAR(50),
    status VARCHAR(20),
    registered_at DATETIME,
    last_online_at DATETIME,
    reserved TEXT
);

-- 项目设备绑定表
CREATE TABLE project_devices (
    id INTEGER PRIMARY KEY,
    project_id INTEGER,
    device_id INTEGER,
    device_name VARCHAR(255),
    device_type VARCHAR(50),
    device_code VARCHAR(100),
    device_ip VARCHAR(50),
    reserved TEXT,
    FOREIGN KEY (project_id) REFERENCES projects(id),
    FOREIGN KEY (device_id) REFERENCES device_registry(id)
);

-- 设备课程绑定表
CREATE TABLE device_courses (
    id INTEGER PRIMARY KEY,
    device_id INTEGER,
    course_id INTEGER,
    course_name VARCHAR(255),
    course_version VARCHAR(20),
    course_package_path VARCHAR(500),
    package_size INTEGER,
    bind_time DATETIME,
    reserved TEXT,
    FOREIGN KEY (device_id) REFERENCES project_devices(id)
);

-- 用户表
CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    username VARCHAR(50) UNIQUE,
    password_hash VARCHAR(255),
    real_name VARCHAR(100),
    role VARCHAR(20),
    status VARCHAR(20) DEFAULT 'active',
    reserved TEXT
);

-- 角色权限表
CREATE TABLE role_permissions (
    role VARCHAR(20),
    permission VARCHAR(50),
    PRIMARY KEY (role, permission)
);

-- 项目版本表
CREATE TABLE project_versions (
    id INTEGER PRIMARY KEY,
    project_id INTEGER,
    version VARCHAR(20),
    name VARCHAR(100),
    description TEXT,
    project_data TEXT,
    status VARCHAR(20),
    created_at DATETIME,
    created_by VARCHAR(50),
    FOREIGN KEY (project_id) REFERENCES projects(id)
);

-- 项目同步记录表
CREATE TABLE project_sync_records (
    id INTEGER PRIMARY KEY,
    local_project_id INTEGER,
    library_project_id INTEGER,
    sync_type VARCHAR(20),
    from_version VARCHAR(20),
    to_version VARCHAR(20),
    sync_time DATETIME,
    success BOOLEAN
);
```
