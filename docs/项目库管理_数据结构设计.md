# 项目库管理系统 - 详细设计方案

## 一、系统概述

本方案用于设计项目库管理系统，支持：
- 项目创建
- 绑定设备和对应课程
- 导入导出项目资源包（加密解密）
- 项目同步版本管理

## 四、功能设计

### 4.1 新建项目

```
1. 输入项目名称
       ↓
2. 选择要绑定的设备
       ↓
3. 为每个设备分配课程
       ↓
4. 保存项目
```

### 4.2 绑定设备和课程

```
项目
├── 设备A
│   └── 课程列表 [课程ID + 版本]
├── 设备B
│   └── 课程列表
└── 设备C
    └── 课程列表
```

### 4.3 设备管理逻辑

```
新建项目时:
└── 从设备资源库查找设备
    ├── 找到 → 绑定
    └── 未找到 → 提示注册或创建

导入项目时:
└── 连接设备资源库
    ├── 设备存在 → 绑定
    └── 设备不存在 → 自动注册 → 绑定
```

## 五、导入/导出设计

### 5.1 加密解密设计

#### 5.1.1 加密流程

```
1. 收集项目+设备+课程数据
       ↓
2. 生成随机AES密钥 (256位)
       ↓
3. 使用AES密钥加密数据
       ↓
4. 使用RSA公钥加密AES密钥
       ↓
5. 输出加密文件
```

#### 5.1.2 解密流程

```
1. 选择加密文件
       ↓
2. 使用RSA私钥解密AES密钥
       ↓
3. 使用AES密钥解密数据
       ↓
4. 解析项目数据
```

### 5.2 导出包结构

```
导出文件
├── metadata.json                     // 元数据
└── data.enc                         // 加密数据
```

### 5.3 导入/导出流程

```
导出流程:
1. 选择项目
       ↓
2. 收集项目+设备+课程数据
       ↓
3. 加密打包
       ↓
4. 输出文件

导入流程:
1. 选择文件
       ↓
2. 解密文件
       ↓
3. 连接设备资源库
    ├── 设备存在 → 绑定
    └── 设备不存在 → 自动注册
       ↓
4. 创建项目并绑定
```

## 六、版本管理

### 6.1 版本标识

语义化版本: MAJOR.MINOR.PATCH

### 6.2 版本管理流程

```
保存版本:
1. 递增版本号
       ↓
2. 保存完整快照
       ↓
3. 更新当前版本

同步:
├── 现场版本 > 项目库版本 → 同步到项目库
├── 现场版本 < 项目库版本 → 从项目库更新
└── 冲突 → 保留双方版本
```

### 6.3 数据表设计

```sql
-- 项目表
CREATE TABLE projects (
    id INTEGER PRIMARY KEY,
    name VARCHAR(255),
    current_version VARCHAR(20) DEFAULT '1.0.0',
    reserved TEXT
);

-- 设备资源库
CREATE TABLE device_registry (
    id INTEGER PRIMARY KEY,
    name VARCHAR(255),
    type VARCHAR(50),
    reserved TEXT
);

-- 项目设备绑定表
CREATE TABLE project_devices (
    id INTEGER PRIMARY KEY,
    project_id INTEGER,
    device_name VARCHAR(255),
    device_type VARCHAR(50),
    reserved TEXT,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);

-- 设备课程绑定表
CREATE TABLE device_courses (
    id INTEGER PRIMARY KEY,
    device_id INTEGER,
    course_id INTEGER,
    course_version VARCHAR(20),
    reserved TEXT,
    FOREIGN KEY (device_id) REFERENCES project_devices(id)
);

-- 用户表
CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    username VARCHAR(50) UNIQUE,
    password_hash VARCHAR(255),
    role VARCHAR(20),
    status VARCHAR(20) DEFAULT 'active',
    reserved TEXT
);

-- 项目版本表
CREATE TABLE project_versions (
    id INTEGER PRIMARY KEY,
    project_id INTEGER,
    version VARCHAR(20),
    project_data TEXT,
    created_at DATETIME,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);
```
