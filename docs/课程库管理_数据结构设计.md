## 三、分数计算规则

### 3.1 计算公式

```
课程总分 = Σ(大步骤分数)

大步骤分数计算：
    如果 is_disqualify = true：
        该大步骤得分 = (任意小步骤丢分) ? 0 : 大步骤总分
    
    如果 is_disqualify = false：
        该大步骤得分 = Σ(小步骤得分)

超时扣分 = floor((实际时长 - 课程时长) / interval_minutes) * deduction_score

最终得分 = 课程总分 - 超时扣分
```

### 3.2 计算示例

```
课程总时长: 600秒
超时规则: 每超过5分钟扣3分

场景1: 实际时长610秒, 课程总分100分
  超时扣分 = floor((610-600)/5) * 3 = 2 * 3 = 6分
  最终得分 = 100 - 6 = 94分

场景2: 大步骤A (is_disqualify=true, 包含3个小步骤各10分)
  - 小步骤1得分10分 → 该大步骤得分30分
  - 小步骤2得分0分 → 该大步骤得分0分(失格)
  - 小步骤3得分10分 → 该大步骤得分0分(失格)
```

## 四、导入/导出设计

### 4.1 加密解密设计

#### 4.1.1 加密流程

```
┌─────────────────────────────────────────────────────────────┐
│                    导出加密流程                              │
├─────────────────────────────────────────────────────────────┤
│  1. 准备课程数据                                            │
│       ↓                                                     │
│  2. 生成随机AES密钥 (256位)                                 │
│       ↓                                                     │
│  3. 使用AES密钥加密 course.json 和资源文件                   │
│       ↓                                                     │
│  4. 使用RSA公钥加密AES密钥                                   │
│       ↓                                                     │
│  5. 打包加密数据到Zip文件                                    │
│       ↓                                                     │
│  6. 输出加密的.zip文件                                      │

```

```

│       ↓                                                     │
│  2. 解压Zip文件                                             │

```

### 4.3 导入流程

#### 4.3.1 非加密文件导入

```
1. 选择Zip文件
       ↓
2. 解压到临时目录
       ↓
3. 验证course.json数据结构
       ↓
4. 解析课程信息并创建课程记录
       ↓
5. 上传resources到服务器 → 获取资源ID
       ↓
6. 替换course.json中的资源路径为资源ID
       ↓
7. 保存到数据库
```

```

       ↓

       ↓

       ↓
4. 使用AES密钥解密data.enc
       ↓
5. 解压解密后的Zip数据
       ↓
6. 验证course.json数据结构
       ↓
7. 解析课程信息并创建课程记录
       ↓
8. 处理资源文件
       ↓
9. 保存到数据库
```

### 4.4 密钥管理

#### 4.4.1 密钥生成

```
RSA密钥对生成:
- 密钥长度: 2048位
- 公钥指数: 65537
- 私钥使用密码保护
```

#### 4.4.2 密钥存储

```
密钥存储策略:
├── 导出时使用: 公钥 (可分发)
├── 导入时使用: 私钥 (严格保管)
└── 密钥备份: 离线存储
```

#### 4.4.3 密钥轮换

```
密钥轮换策略:
├── 定期更换: 建议每6个月更换一次
├── 紧急更换: 密钥泄露时立即更换
├── 旧版本兼容: 保留历史公钥用于解密旧文件
└── 轮换记录: 记录所有密钥更换历史
```

### 5.3 版本保存流程

```
1. 用户点击"保存新版本"或"发布"
       ↓
2. 获取当前版本号 (如: 1.2.0)
       ↓
3. 递增版本号
   - PATCH+1: 1.2.0 → 1.2.1 (常规保存)
   - MINOR+1: 1.2.0 → 1.3.0 (新增功能)
   - MAJOR+1: 1.2.0 → 2.0.0 (重大变更)
       ↓
4. 序列化完整课程数据为JSON
       ↓
5. 保存到course_versions表
       ↓
6. 更新courses表的current_version
       ↓
7. 记录版本保存成功
```

### 5.5 同步策略

```
┌─────────────────────────────────────────────────┐
│               版本同步流程                       │
├─────────────────────────────────────────────────┤
│                                                 │
│  现场版本(v1.2.0)  ←→  课程库版本(v1.2.0)     │
│       │                       │                  │
│       │  检测版本差异          │                  │
│       ↓                       ↓                  │
│  ┌─────────────────────────────────────────┐    │
│  │  版本对比结果:                            │    │
│  │  - 现场 > 课程库: 提示"现场有更新"       │    │
│  │  - 现场 < 课程库: 提示"课程库有新版本"    │    │
│  │  - 现场 = 课程库: 无需操作                │    │
│  └─────────────────────────────────────────┘    │
│                                                 │
│  同步选项:                                       │
│  - [同步到现场]   拉取课程库版本覆盖本地         │
│  - [同步到课程库] 推送本地版本到课程库           │
│  - [另存为]      保存为新版本(保留双方)          │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 5.6 冲突处理

```
场景：现场v1.2.0 和 课程库v1.2.0 都进行了修改

处理方案：
1. 每次保存都生成新版本号
2. 两个版本都保留，不覆盖
3. 用户可以选择：
   - 查看并使用现场版本
   - 查看并使用课程库版本
   - 手动合并内容后保存为全新版本
```

### 5.7 离线处理

```
网络状态检测：
├── 有网络时：启用同步功能
└── 无网络时：
    ├── 禁止拉取课程库（只读模式）
    ├── 禁止推送到课程库
    ├── 本地版本正常管理
    └── 恢复网络后提示同步
```
